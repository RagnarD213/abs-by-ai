**Business goal this serves:** App adoption (iOS App Store submission) → profitability

## Objective

Build the existing iOS Capacitor wrapper (`ios-app/`), run it in the iOS Simulator, and walk through the ENTIRE product inside the wrapper to confirm everything works and looks right on iPhone before the App Store submission. The wrapper was built in June; every major feature shipped since (Trainer, Nutritionist, Macro Tracker, Sleep Coach, Supplement Audit, Progress Log, Coach Brief, Transformations, member hub, pre-trial quiz, proof banner) has NEVER been exercised inside it. The output is a pass/fail feature checklist, screenshots for the App Store listing, and fixes for any wrapper-specific breakage found.

## Detailed Plan

If Xcode is still missing, STOP and tell Dan exactly what to do (upgrade macOS via System Settings, install Xcode from the App Store, then run `sudo xcode-select -s /Applications/Xcode.app`, `sudo xcodebuild -license accept`, `xcodebuild -downloadPlatform iOS`). 2. **Build:** `cd ios-app && npx cap sync ios && xcodebuild -project ios/App/App.xcodeproj -scheme App -destination 'platform=iOS Simulator,name=iPhone 17' build` (adjust device name to an installed simulator; `xcrun simctl list devices`). Fix any Capacitor-8/SPM drift the sync surfaces.
3. **Run in simulator with the live panel open** (Claude Code's iOS Simulator tools: attach first, then launch the built .app) so Dan can watch. Confirm cold launch: splash → status bar → site loads → safe areas correct (no content under the notch/home indicator).

   - Physical print flow: product selector opens and reaches Stripe checkout UI (do NOT complete a purchase).
   - Member features (use a comp/test account): pre-trial quiz, member hub, Daily Coach Brief, AI Trainer (incl. weekday workout naming), AI Nutritionist, Macro Tracker (multi-photo + meal prep + leftovers — camera permission prompt), Sleep Coach, Supplement Audit (async job survives app backgrounding?), Progress Log + weigh-in, My Transformations gallery, password reset email flow.
   - Account deletion (if shipped by then).
   - Keyboard behavior in forms (Capacitor Keyboard plugin), pull-to-refresh/back gestures, dark mode, and rotation lock.
5. **Fix what breaks.** Expected fix locations: the native-integration script in `public/index.html` (deploys via Railway, no rebuild needed) vs. `capacitor.config.json`/plugins/Info.plist (needs rebuild). Prefer web-side fixes.
6. **App Store screenshots:** required sizes 6.9" (iPhone 16 Pro Max class) and 6.5". `xcrun simctl io booted screenshot shot.png`. Capture the strongest screens: before/after result, Trainer workout, Coach Brief, Macro Tracker.
7. **Deliverables:** checklist of pass/fail per feature with screenshots, list of fixes shipped (each committed + pushed + live-verified per project workflow), screenshots folder for the listing, and an updated `AI_COORDINATION.md`. Flag anything that should go in App Store Review Notes (physical-goods-only payments, web-membership sign-in, minor-refusal guard, native features list).

## Things to Avoid / Lessons Learned

- The app loads the LIVE site — web fixes reach the app on Railway deploy, no rebuild. Only native-layer changes (plugins, config, icons, Info.plist) need a rebuild. Don't rebuild reflexively.
- Do not complete any real Stripe payment while testing; stop at the checkout screen.
- Local dev server has dummy AI keys — the simulator should point at production absbyai.com (it does by default via `capacitor.config.json`); don't repoint it at localhost expecting AI features to work.
- Xcode-toolchain fixes (xcode-select, license, macOS upgrade) need Dan's password — surface them immediately with exact commands instead of retrying.
- Past coordinate-click quirks: on some pages the screenshot/viewport mapping of coordinate clicks was off — prefer accessibility-tree/DOM-based interaction or direct `.click()` when a tap seems to miss.
